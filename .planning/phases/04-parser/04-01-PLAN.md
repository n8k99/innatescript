Implement the following functions in `src/parser/parser.lisp`:

1. **`parse-statement-list`** (replace stub from Task 1): Loop while cursor has tokens. Skip `:newline` tokens. For each non-newline token, call `parse-statement`. Collect results in a list, return `(nreverse stmts)`.

2. **`parse-statement`**: Dispatch on `(token-type (cursor-peek cursor))`:
   - `:prose` ظْ consume, return `(make-node :kind +node-prose+ :value (token-value tok))`
   - `:hash` ظْ call `parse-heading`
   - `:lbracket` ظْ call `parse-bracket`
   - `:arrow` ظْ call `parse-emission` (stub: just consume and return nil for now ظ¤ Plan 03)
   - `:decree` ظْ call `parse-decree` (stub for now ظ¤ Plan 02)
   - Otherwise ظْ call `parse-expression`, then check for `:pipe-pipe` ظْ call `parse-fulfillment` (both stubs for now)
   After getting the statement node, check if next token is `:arrow` ظ¤ if so, wrap in emission (left-associative, handled in Plan 03). Check if next token is `:pipe-pipe` ظ¤ if so, wrap in fulfillment (handled in Plan 03).

3. **`parse-expression`**: Dispatch on token type:
   - `:at` ظْ stub returning nil (Plan 02: parse-reference)
   - `:lparen` ظْ stub returning nil (Plan 02: parse-agent)
   - `:lbrace` ظْ stub returning nil (Plan 02: parse-bundle-or-lens)
   - `:bang-bracket` ظْ stub returning nil (Plan 02: parse-search)
   - `:lbracket` ظْ call `parse-bracket`
   - `:hash` ظْ call `parse-heading`
   - Otherwise ظْ call `parse-atom`

4. **`parse-atom`**: Dispatch on token type:
   - `:bare-word` ظْ consume, return `(make-node :kind +node-bare-word+ :value (token-value tok))`
   - `:string` ظْ consume, return `(make-node :kind +node-string-lit+ :value (token-value tok))`
   - `:number` ظْ consume, return `(make-node :kind +node-number-lit+ :value (token-value tok))`
   - `:wikilink` ظْ consume, return `(make-node :kind +node-wikilink+ :value (token-value tok))`
   - `:emoji-slot` ظْ consume, return `(make-node :kind +node-emoji-slot+ :value (token-value tok))`
   - Otherwise ظْ signal `innate-parse-error` with token line/col, text "Unexpected token TYPE"

5. **`parse-bracket`**: Consume `:lbracket`. Peek: if next is `:bare-word` and the token after that is NOT `:colon` (use cursor-peek-next), consume the bare-word as the bracket's name (value). Call `parse-bracket-body` to get children. Return `(make-node :kind +node-bracket+ :value name :children children)`.

   Special case: if next after lbracket is `:bare-word` and peek-next IS `:colon`, do NOT consume as name ظ¤ it's a kv-pair, fall through to bracket-body parsing.

6. **`parse-bracket-body`**: Loop until `:rbracket` or EOF. Dispatch:
   - `:rbracket` ظْ consume, return
   - `:newline` ظْ consume, skip
   - `:prose` ظْ consume, push prose node
   - `:bare-word` with peek-next `:colon` ظْ call `parse-kv-pair`
   - `:lbracket` ظْ call `parse-bracket` (recursive)
   - `:hash` ظْ call `parse-heading`
   - `:at` ظْ call `parse-expression` (stub returns nil, Plan 02 fills in)
   - `:bang-bracket` ظْ call `parse-expression`
   - `:arrow` ظْ call `parse-expression`
   - `:decree` ظْ call `parse-expression`
   - Otherwise ظْ call `parse-expression`
   On EOF (nil token): signal `innate-parse-error` line 0 col 0 text "Unterminated bracket expression"

7. **`parse-kv-pair`**: Consume bare-word (key). Consume `:colon` via cursor-expect. Parse value via `parse-expression`. Return `(make-node :kind +node-kv-pair+ :value key-name :children (list value-node))`.

8. **`parse-heading`**: Consume `:hash`. If next is `:bare-word`, consume and accumulate words until non-bare-word. Join with spaces. Return `(make-node :kind +node-heading+ :value heading-text)`. If next is `:lbracket`, the heading has a bracket child ظ¤ consume text then parse bracket as child.

All helper functions for expression types not yet implemented (reference, agent, bundle, search, decree) should be stubs that signal `innate-parse-error` with text "Not yet implemented: TYPE". These will be filled in by Plan 02.

Add tests in `tests/test-parser.lisp` for all behaviors listed above. Use `(parse (tokenize "source"))` as the test entry point ظ¤ the parser receives real token streams.
