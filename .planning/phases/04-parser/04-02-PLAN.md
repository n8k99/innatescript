  <action>
Replace the stubs in `parse-expression` for `:at`, `:lparen`, `:lbrace`, `:bang-bracket` dispatch cases, and add modifier handling. Implement:

1. **`parse-reference`**: Consume `:at`. Expect `:bare-word` for the name. Initialize empty `children` list and `props` plist.

   **Qualifier chain** — if next token is `:colon`:
   - Consume the colon
   - If next token is `:string`, consume it. Qualifier value = the string's value. Push `(make-node :kind +node-string-lit+ :value qual-val)` to children. Set `(setf props (list :qualifiers (list qual-val)))`.
   - If next token is `:bare-word`, accumulate bare-words: loop while peek is `:bare-word` (and not followed by `:colon` at bracket-body level — but qualifier context means we just accumulate). Stop at `:plus`, `:lbrace`, `:rbracket`, `:rparen`, `:rbrace`, `:newline`, `:pipe-pipe`, `:arrow`, `:comma`, or nil. Join accumulated words with spaces. Push as `+node-string-lit+` child. Set props.

   **Combinator** — if next token is `:plus`:
   - Consume `:plus`
   - Expect `:bare-word` for combinator name
   - Push `(make-node :kind +node-combinator+ :value comb-name)` to children
   - Set `(setf props (list* :combinator comb-name props))`

   **Lens** — if next token is `:lbrace`:
   - Call `parse-lens` and push result to children

   Return `(make-node :kind +node-reference+ :value name :children (nreverse children) :props props)`.

2. **`parse-agent`**: Consume `:lparen`. Expect `:bare-word` for agent name. Expect `:rparen`. Return `(make-node :kind +node-agent+ :value agent-name)`.
   Note: `(agent){instruction}` is handled by adjacency — the agent and bundle are siblings in the parent's children list. No special commission node.

3. **`parse-bundle-or-lens`**: Consume `:lbrace`. Peek at next two tokens:
   - If `:bare-word` followed by `:colon` → this is a lens. Call `parse-lens-body` (parse kv-pairs until `:rbrace`). Return `(make-node :kind +node-lens+ :children kv-pairs)`.
   - If `:bare-word` followed by `:rbrace` → this is a bundle. Consume bare-word, consume rbrace. Return `(make-node :kind +node-bundle+ :value bundle-name)`.
   - Otherwise → treat as lens with expression children.

4. **`parse-lens`**: Consume `:lbrace`. Parse kv-pairs until `:rbrace`. Each kv-pair: consume bare-word (key), expect `:colon`, parse expression (value — typically bare-word like `==`). Consume `:rbrace`. Return `(make-node :kind +node-lens+ :children kv-pairs)`.

5. **`parse-search`**: Consume `:bang-bracket`. Parse expression list until `:rbracket` — each element via `parse-expression`. Consume `:rbracket`. Return `(make-node :kind +node-search+ :children exprs)`.

6. **`parse-modifier`**: Consume `:slash`. Expect `:bare-word`. Return `(make-node :kind +node-modifier+ :value modifier-name)`.

7. Update `parse-expression` to handle `:slash` → call `parse-modifier`.

8. After any expression in `parse-statement`, check if next token is `:slash` → if so, the modifier attaches to the preceding expression. Wrap: the modifier becomes a child of the expression, or store as a prop. Per CONTEXT.md: presentation modifier "attaches to preceding expression." Implementation: after getting the expression node from parse-expression, if next is `:slash`, parse modifier and set it as a prop on the expression node: `(setf (node-props expr) (list* :modifier modifier-value (node-props expr)))`.

Add all tests listed in behavior section to `tests/test-parser.lisp`. Test the compound reference thoroughly — verify node-kind, node-value, children kinds, and props values.
