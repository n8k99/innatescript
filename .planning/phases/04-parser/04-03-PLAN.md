Refactor `parse-statement` to implement the full precedence chain. The statement-level parsing becomes:

1. **`parse-statement`** refactored as a precedence climb:
   - Call `parse-fulfillment-expr` (loosest binding)
   - Return the result

2. **`parse-fulfillment-expr`** (PAR-12):
   - Call `parse-emission-expr` to get left side
   - Loop: while next token is `:pipe-pipe`:
     - Consume `:pipe-pipe`
     - Call `parse-emission-expr` for right side
     - Wrap: `left = (make-node :kind +node-fulfillment+ :children (list left right))`
   - Return left
   - This gives left-associativity: `a || b || c` => `(|| (|| a b) c)`

3. **`parse-emission-expr`** (PAR-13):
   Two cases:
   a) **Leading `->` (emission at statement start):**
      - If current token is `:arrow`, consume it
      - Parse value via `parse-expression`
      - While next token is `:comma`, consume and parse another value
      - Return `(make-node :kind +node-emission+ :children values)`
   b) **Infix `->` (emission after expression):**
      - Call `parse-expression` for left side
      - Loop: while next token is `:arrow`:
        - Consume `:arrow`
        - Parse value via `parse-expression`
        - While next token is `:comma`, consume and parse another value
        - Wrap: `left = (make-node :kind +node-emission+ :children (cons left values))`
      - Return left
   - Left-associativity: `a -> b -> c` => `(-> (-> a b) c)`

4. Keep the existing `parse-statement` top-level dispatch for `:prose`, `:hash`, `:decree`, `:lbracket` which bypass the precedence chain (they are statement-level constructs, not expressions). For everything else, route through `parse-fulfillment-expr`.

   Updated dispatch in `parse-statement`:
   - `:prose` → consume, return prose node (no operator attachment)
   - `:hash` → parse-heading (no operator attachment)
   - `:decree` → parse-decree (no operator attachment)
   - `:newline` → should not reach here (consumed by statement-list)
   - Everything else → `parse-fulfillment-expr`

5. Inside bracket bodies, expressions also need to support `->` and `||`. Update `parse-bracket-body` to call `parse-fulfillment-expr` instead of `parse-expression` for the general case. Keep the kv-pair and prose special cases.
