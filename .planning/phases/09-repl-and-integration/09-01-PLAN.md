2. Implement `src/repl.lisp` with two exported functions:

   `(defun run-file (path env))`
   - Read entire file with `(with-open-file ...)` into a string
   - Call `(evaluate (parse (tokenize source)) env)`
   - Return the list of results

   `(defun repl (&optional env))`
   - If env is nil, create one: `(make-eval-env :resolver (make-stub-resolver))`
   - Print a short welcome line: `"Innate v0.1.0"` then newline
   - Loop:
     - Print `"innate> "` prompt (use `format t` then `finish-output`)
     - Read line with `read-line` — on EOF (second return value T), print newline and return
     - Check quit commands: if input is `"(quit)"`, `"(exit)"`, or `":quit"`, return
     - Skip empty/whitespace-only lines (continue loop)
     - Wrap eval pipeline in `handler-case`:
       - Try: `(let ((results (evaluate (parse (tokenize line)) env))) ...)`
       - Print each result: if `resistance-p`, print `"= [resistance] message"`;
         if `innate-result-value`, print `"= value"`;
         if string, print `"= string"`;
         otherwise print `"= ~S"` with `format`
       - Catch `innate-parse-error (e)`: print `"Parse error at line ~D, col ~D: ~A"` using `parse-error-line`, `parse-error-col`, and the condition's report
       - Catch `innate-resistance (e)`: print `"[commission queued] ~A"` using `resistance-condition-source`
       - Catch `error (e)`: print `"Error: ~A"` using the condition
     - Loop back to prompt

   Important: The REPL reuses the SAME env across iterations so decrees accumulate. The `handler-case` wraps ONLY the eval pipeline, not the read-line.
  </action>
