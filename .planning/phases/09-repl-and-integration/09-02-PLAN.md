Create `run-repl.sh` at the project root. Follow the same patterns as `run-tests.sh` for ASDF loading.

```bash
#!/usr/bin/env bash
# run-repl.sh — start the Innate REPL or evaluate a .dpn file
# Usage:
#   ./run-repl.sh              — start interactive REPL
#   ./run-repl.sh file.dpn     — evaluate file and exit
#   rlwrap ./run-repl.sh       — REPL with line editing

set -e

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
```

Two modes:

**File mode** (argument given): Use `sbcl --non-interactive` to load the system, call `(innate.repl:run-file "<filepath>" (innate.eval.resolver:make-eval-env :resolver (innate.eval.stub-resolver:make-stub-resolver)))`, print each result, then exit. The file path argument should be resolved relative to SCRIPT_DIR if not absolute (use `realpath` or prepend SCRIPT_DIR). Print results using a simple loop — for each result, if it's a string print it, if it's an innate-result print its value, if it's a resistance print "[resistance] message". Exit code 0 on success.

**Interactive mode** (no argument): Use `sbcl` (NOT --non-interactive) to load the system, then call `(innate.repl:repl)`. This gives SBCL control of stdin so read-line works. When repl returns (user quit), call `(sb-ext:exit :code 0)`.

Both modes:
- `--eval "(require :asdf)"`
- `--eval "(push #p\"${SCRIPT_DIR}/\" asdf:*central-registry*)"`
- `--eval "(asdf:load-system :innatescript)"` (suppress output with `--eval "(let ((*standard-output* (make-broadcast-stream))) (asdf:load-system :innatescript))"` so compilation messages don't pollute the REPL)
- `--noinform` flag to suppress the SBCL banner

Make the script executable: `chmod +x run-repl.sh`

IMPORTANT: For interactive mode, do NOT use `--non-interactive` — that disables read-line from stdin. Use `--script` or just `sbcl` with `--noinform` flags. The key difference: file mode needs `--non-interactive` (exit after eval), interactive mode needs the SBCL process to stay alive for read-line.
